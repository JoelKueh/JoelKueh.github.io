JChessV2

JChessV2 is a C++ remake of a Chess game I made in C# called JChess. The project was designed to be the base of a Chess AI that I will start writing soon. As a chess AI needs to look through hundreds of millions of positions every time it makes a decision. I worked very hard to make sure that move generation would be as fast as possible. The move generation algorighms make use of lookup tables and bithacks to efficiently generate moves. After multiple iterations and hous upon hours of reading move generation theory on ChessProgramming.org, I managed to search through chess positions at 43 million nodes per second on a single thread.

Currently, the game can only be played through a terminal UI, but I'm leanrning the OpenGL api to add an optional GUI as well. I'm also working with a couple of friends to add networking to the game so that users can play online rather than just locally. Eventually, my friends and I will move on to writing the AI itself, but that can't come without a decent ammount of prep work and a lot more reading. You can take a look at what we've got so far on my github in the link on the right.

Mandelbrot Set

In the spring of my senior year of high school, I really took an interest in the Mandelbrot Set. If you haven't heard of it, it's a fractal produced by a relatively simple recursive algorithm. If the algorithm converges for a given point on the complex plane, then you plot a black point, if the algorighm diverges for a given point, you plot a white point (in reality, most people plot a color based on the number of iterations it took for the result to reach some escape radius). The set of all points where the algorithm converges is the Mandelbrot Set.

As this algorithm has to once on every pixel on the screen individually. I thought it would be interesting to learn how pass this computation off to the GPU for efficiency. As a result, I decided to write this program with the DirectX API. At the moment, this program just renders one frame of the Mandelbrot Set. You can't interact with it, or zoom in to different sections of it. I have plans to add to this program and fix this in the future.

University Kernel Object (U.ko)

Since Fall 2022, I've been working with a group called the University Kernel Object. In essence, we are a group of UMNTC students who are really passionate about the Linux Kernel. Led by a couple of University Professors as well as a Kernel maintainer, we have been exploring different aspects of the Linux Kernel and learning about Kernel development.

In particular, I have been involved with the Virtual Memory Subgroup. Over the spring of 2023, we searched through kernel source code to learn how virtual memory was implemented in the kernel. In the end, we wrote a program which could map the virtual address space of a running program to its physical counterpart.
